1. Key Information to Gather
-----Technical
User authentication: Registration, login, password reset.
Receipt generation: Unique receipt numbers, date/time, customer info, itemized list, totals, taxes, payment method.
Database models: Users, Customers, Receipts, Items/Products, Payments.
PDF/Printable receipts: Use libraries like WeasyPrint, xhtml2pdf, or ReportLab. (IronPDF or Playwright found recommended due to html to pdf conversion)
Email integration: Send receipts to customers.
Admin dashboard: Manage users, view receipts, analytics.
API endpoints: For integration with other systems (optional).
Security: Data protection, HTTPS, user permissions.
Responsive UI: Works on desktop and mobile.
-----Non-Technical
Target users: Small businesses, freelancers, shops.
User journey: How users sign up, create receipts, send/download them.
Compliance: Local tax laws, data privacy.
Branding: Logo, color scheme, business info on receipts. ['FBFBFB', '31081F', '084B83', '4A5240', 'F4E409']
Support: Help pages, contact info.
